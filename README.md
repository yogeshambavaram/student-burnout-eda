# The Danger of Blind Data: An Exploratory Data Analysis on Student Burnout

## 📌 Project Overview
This project began as a standard Exploratory Data Analysis (EDA) on a 150,000-row Kaggle dataset regarding student burnout, sleep, and screen time. The initial goal was to clean the data, visualize trends, and prepare the dataset for predictive machine learning models. 

However, during the visualization phase, the project pivoted from standard EDA to an investigation of **Data Integrity**. By analyzing distribution densities and calculating a Pearson correlation matrix, I discovered that the dataset was entirely synthetic. 

## 🛠️ Tech Stack & Tools
* **Environment:** Google Colab / Jupyter Notebook
* **Language:** Python 3
* **Libraries:** * `pandas` (Data manipulation and cleaning)
  * `matplotlib` & `seaborn` (Data visualization and heatmap generation)

## 📊 Key Findings
1. **Uniform Distributions:** Addressed severe overplotting of 150,000 data points by utilizing a 2D density heatmap (`sns.histplot`). The results showed an unnatural, near-uniform distribution with no real-world behavioral clustering.
2. **Zero Correlation:** A Pearson correlation matrix (`sns.heatmap`) revealed a correlation coefficient of nearly zero ($r \approx 0$) across all variable pairs, mathematically confirming the data was artificially generated by an algorithm rather than collected from human subjects.

## 💡 Lessons Learned
This project reinforced a critical lesson in Data Science: **writing functional code is only half the job.** Building machine learning models on unverified, synthetic data leads to flawed conclusions. True data science requires critically questioning the integrity of the data itself before applying complex algorithms.
